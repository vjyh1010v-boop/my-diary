# 구글·카카오 소셜 로그인 설정 가이드

`social-login.html`이 실제로 동작하려면 아래 준비가 필요합니다. 소셜 로그인은 구글/카카오 서버와 통신해야 하므로, 파일을 더블클릭해 여는 방식(file://)에서는 동작하지 않고 **반드시 인터넷 주소(https://)가 있어야** 합니다.

---

## 0단계. 먼저 알아둘 것

- 소셜 로그인은 "누가 들어오는지 확인하는 문지기" 역할입니다.
- 데이터 자체는 여전히 각 기기의 브라우저에 저장됩니다. 데이터까지 서버에 안전하게 보관·동기화하려면 Firebase Firestore 같은 백엔드가 추가로 필요합니다 (원하시면 다음 단계로 도와드릴 수 있어요).
- 지금 diary.html에 이미 넣어드린 **비밀번호 잠금 + 암호화**는 호스팅 없이도 동작하는 실제 보안장치이니, 소셜 로그인 준비 전까지는 이것을 사용하세요.

## 1단계. 무료 호스팅에 올리기 (GitHub Pages 기준)

1. https://github.com 가입 → 우측 상단 `+` → **New repository** → 이름 예: `my-diary` → Public → Create
2. **Add file > Upload files**로 `diary.html`, `diary.css`, `diary.js`, `social-login.html` 4개 업로드 → Commit
3. 저장소의 **Settings > Pages** → Branch를 `main`으로 선택 → Save
4. 몇 분 뒤 `https://내아이디.github.io/my-diary/` 주소가 생깁니다.
   - 로그인 페이지: `https://내아이디.github.io/my-diary/social-login.html`

## 2단계. 구글 로그인 (Firebase)

1. https://console.firebase.google.com → **프로젝트 추가** (이름 자유)
2. 왼쪽 메뉴 **빌드 > Authentication** → 시작하기 → **로그인 방법** 탭 → **Google** 사용 설정
3. **Authentication > 설정 > 승인된 도메인**에 `내아이디.github.io` 추가
4. 프로젝트 개요 옆 ⚙ → **프로젝트 설정** → 아래 "내 앱" → 웹 앱(</>) 등록 → **SDK 구성** 값 복사
5. `social-login.html`의 **[여기에 붙여넣기 1]** 부분에 apiKey, authDomain, projectId, appId를 붙여넣기

## 3단계. 카카오 로그인 (Kakao Developers)

1. https://developers.kakao.com → 로그인 → **내 애플리케이션 > 애플리케이션 추가하기**
2. **앱 키** 메뉴에서 `JavaScript 키` 복사 → `social-login.html`의 **[여기에 붙여넣기 2]**에 붙여넣기
3. **플랫폼 > Web 플랫폼 등록** → 사이트 도메인에 `https://내아이디.github.io` 입력
4. **제품 설정 > 카카오 로그인** → 활성화 ON
   - Redirect URI: `https://내아이디.github.io/my-diary/social-login.html`
5. **동의항목**에서 '닉네임' 항목을 필수 동의로 설정

## 4단계. 다이어리에 로그인 문 달기 (선택)

로그인을 안 하면 다이어리를 못 열게 하려면, `diary.js`의 `function boot(){` 바로 **아랫줄**에 다음 3줄을 추가하세요:

```js
  // 소셜 로그인을 하지 않았으면 로그인 페이지로 보내기
  if(location.protocol.indexOf('http')===0 && !sessionStorage.getItem('diaryLogin')){
    location.href='social-login.html'; return;
  }
```

- `sessionStorage`는 브라우저 탭을 닫으면 사라지므로, 다시 열면 재로그인이 필요합니다(자동로그인은 각 SDK의 토큰 유지 기능으로 확장 가능).
- 이 조건은 http(s)로 열었을 때만 작동하므로, PC에서 파일로 직접 열 때는 로그인 없이 열립니다(개발 편의).

## 자주 묻는 문제

| 증상 | 원인/해결 |
|---|---|
| 구글 팝업이 바로 닫힘 | Firebase 승인된 도메인에 내 주소가 없음 → 2-3단계 확인 |
| 카카오 "KOE101" 오류 | JavaScript 키가 아니라 REST 키를 넣었거나, Web 플랫폼 도메인 미등록 |
| 파일로 열면 로그인 버튼이 안 됨 | 정상입니다. 소셜 로그인은 https 주소에서만 동작해요 |
| 로그인은 되는데 데이터가 기기마다 다름 | 정상입니다. 데이터 동기화는 별도 백엔드(Firestore 등)가 필요해요 |
