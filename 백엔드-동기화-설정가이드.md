# 클라우드 동기화(백엔드) 설정 가이드

`diary-sync.js`가 담당하는 기기 간 동기화(PC ↔ 폰) 기능을 켜는 방법입니다. 백엔드로는 구글의 무료 서비스 **Firebase**를 사용합니다 (서버를 직접 만들거나 돈 낼 필요 없음, 무료 한도가 개인 사용에 충분).

**보안 핵심**: 다이어리에 🔒 비밀번호 잠금을 켜두면, 클라우드에는 암호화된 덩어리만 올라가서 구글조차 내용을 읽을 수 없습니다 (종단간 암호화). 꼭 비밀번호 잠금과 함께 사용하세요.

---

## 1단계. 호스팅에 올리기 (아직 안 했다면)

동기화는 인터넷 통신이 필요해 파일로 여는 방식(file://)에서는 동작하지 않습니다.

1. https://github.com 가입 → `+` → **New repository** → 이름 예: `my-diary` → Public → Create
2. **Add file > Upload files** → `diary.html`, `diary.css`, `diary.js`, `diary-sync.js` 업로드 → Commit
3. **Settings > Pages** → Branch `main` 선택 → Save
4. 몇 분 뒤 생기는 주소로 접속: `https://내아이디.github.io/my-diary/diary.html`

## 2단계. Firebase 프로젝트 만들기

1. https://console.firebase.google.com → **프로젝트 추가** (이름 자유, 애널리틱스는 꺼도 됨)
2. 프로젝트 개요 옆 ⚙ → **프로젝트 설정** → 아래 "내 앱" → **웹 앱(</>)** 등록
3. 나오는 **SDK 구성**(apiKey, authDomain, projectId, appId)을 복사해서
   `diary-sync.js` 파일의 **[여기에 붙여넣기]** 부분에 붙여넣기 → GitHub에 다시 업로드

## 3단계. 로그인 기능 켜기 (Authentication)

1. Firebase 콘솔 왼쪽 **빌드 > Authentication** → 시작하기
2. **로그인 방법** 탭 → **이메일/비밀번호** → 사용 설정 → 저장
3. **설정 > 승인된 도메인**에 `내아이디.github.io` 추가
   - (차후 구글/카카오 로그인을 추가할 때도 이 같은 프로젝트를 그대로 쓰면 됩니다)

## 4단계. 데이터 저장소 만들기 (Firestore)

1. 왼쪽 **빌드 > Firestore Database** → **데이터베이스 만들기** → 위치 기본값 → **프로덕션 모드**로 시작
2. **규칙** 탭에 아래 내용을 통째로 붙여넣고 **게시**:

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // 각 사용자는 자기 문서(diaries/자기UID)만 읽고 쓸 수 있음
    match /diaries/{uid} {
      allow read, write: if request.auth != null && request.auth.uid == uid;
    }
  }
}
```

이 규칙 덕분에 로그인한 본인 외에는 아무도 내 데이터에 접근할 수 없습니다.

## 5단계. 사용 방법

1. PC에서 다이어리 열기 → 상단 **☁ 동기화** 버튼 → **회원가입** (이메일/비밀번호)
   - 이 계정 비밀번호는 다이어리 잠금 비밀번호와 별개입니다. 같게 하지 않는 것을 추천!
2. **⬆ 클라우드에 업로드** 클릭
3. 폰에서 같은 주소 접속 → ☁ 동기화 → 같은 계정으로 **로그인** → **⬇ 클라우드에서 다운로드**
4. 🔒 잠금을 쓰는 중이면, 다운로드 후 PC와 같은 다이어리 비밀번호로 잠금 해제

이후에는 "수정한 기기에서 업로드 → 다른 기기에서 다운로드" 순서만 기억하면 됩니다.

## 자주 묻는 문제

| 증상 | 원인/해결 |
|---|---|
| ☁ 버튼 클릭 시 "설정이 필요해요" | diary-sync.js에 Firebase 키를 아직 안 붙여넣음 (2단계) |
| 로그인 시 오류만 나옴 | Authentication에서 이메일/비밀번호 사용 설정을 안 켬 (3단계) |
| 업로드 시 권한 오류 | Firestore 규칙을 안 붙여넣었거나 게시를 안 함 (4단계) |
| 파일로 열면 ☁ 버튼이 안내만 나옴 | 정상. 동기화는 호스팅 주소(https)에서만 동작 (1단계) |
| 다운로드 후 비밀번호를 물어봄 | 정상. 업로드한 기기와 같은 다이어리 비밀번호를 입력하면 열림 |
